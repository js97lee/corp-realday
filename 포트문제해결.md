# 포트 5001 문제 해결 가이드

## 문제 상황
포트 5001에서 서버가 실행되지 않는 경우

## 해결 방법

### 1. 포트 5001이 사용 중인지 확인

터미널에서:
```bash
lsof -ti:5001
```

프로세스가 나오면:
```bash
# 프로세스 종료
lsof -ti:5001 | xargs kill -9
```

### 2. 서버 실행

```bash
cd /Users/jisulee/A.홈페이지/server
npm run dev
```

성공하면:
```
서버가 포트 5001에서 실행 중입니다.
http://localhost:5001
```

### 3. 다른 포트 사용하기

포트 5001이 계속 문제가 있으면 다른 포트(예: 5002)를 사용할 수 있습니다:

**server/server.js 파일 수정:**
```javascript
const PORT = process.env.PORT || 5002
```

**src/utils/api.js 파일 수정:**
```javascript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:5002/api'
```

### 4. 서버가 실행되는지 확인

브라우저에서 접속:
- `http://localhost:5001/api/contacts`

또는 터미널에서:
```bash
curl http://localhost:5001/api/contacts
```

정상이면 JSON 데이터가 반환됩니다.

### 5. 프론트엔드와 연결 확인

1. 프론트엔드 서버 실행: `npm run dev` (포트 3000)
2. 백엔드 서버 실행: `cd server && npm run dev` (포트 5001)
3. 브라우저에서 `http://localhost:3000/admin` 접속
4. 로그인 테스트

## 일반적인 오류

### "Cannot find module"
→ `cd server && npm install` 다시 실행

### "EADDRINUSE"
→ 포트가 사용 중입니다. 위의 1번 방법으로 해결

### "CORS 오류"
→ 백엔드 서버가 실행되지 않았습니다. 백엔드 서버를 먼저 실행하세요.
